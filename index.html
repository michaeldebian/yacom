                    Yacom (Yet Another Container Orchestrator Manager)

The purpose of YACOM is to aid developers to setup developer environments on their laptops. It will allow developers to easily build docker images (CentOS, Ubuntu and SmartOS), create/destroy instances, deploy in parallel to hundreds of servers, add a preconfigured nginx proxy server for your ruby,nodejs application . The vision is to allow a single point of management for DEVOPS/Software Engineers which will allow them to add/remove containers on the fly, allowing developers to focus 100% on the code, also, it will add/remove instances to/from load balancers.

                             YACOM Architecture
  • Proxy server (nginx)
  • Rsync image repo with docker nodes
  • Easy to build images on demand
  • Reporting capabilities for management per charge code (department)
  • Reporting for admins for resource utilization (pool management)

  • Cassandra based

                         What will YACOM's contain?

  • Spin up/down containers.

  • Generate security keys.
  • Edit/save YAML files online.
  • Support for running different command in languages such us shell, powershell, node, python, ruby, java, among others.
  • Be able to predefined actions with a push of a button (restart apps, restart process, perform an action if process is not running).

CI/CD

  • Integration with github to release new code and rollback.
  • Rewrite it in nodejs enhancing https://github.com/shipitjs/shipit-deploy
  • Integration with bit torrent based releases.

Sysadmin

  • Scheduling mechanism to schedule tasks and notify admins via email, sms.
  • Autonomous, which will migrate guests among hosts.- SSH into multiple machines in parallel.
  • Server Taxonomy to group servers by location, application, cluster, etc.
  • Execute commands to windows and linux servers

  • Ability to create predefined templates for server provisioning.

                                   Containers

  • YACOM seeks to provide an API interface to kubernetes as well as github, making it easy for developers to obtain resources within the enterprise.

  • It will apply the best practices in case of failure, by either doing it autonomic ally or consulting the sysadmin.
  • Autoscaling capabilities based on rules predefined by admin.
