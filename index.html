                        ** Yacom (Yet Another Container Openstack Manager) **

The purpose of YACOM is to provide an alternative to Openstack, but in a much simplified manner. YACOM seeks to integrate with KVM and the container technology in an API format. The vision is to allow the average system administrator/Devops engineer to have a ready to work platform in the matter of minutes.

                            ** What will YACOM's contain? **
  • Login platform for users (developers to create servers), for admins to change resource settings.
  • Integration with CI/CD technologies facilitating a ready to go solution.
  • SSH into multiple machines in parallel.
  • Server Taxonomy to group servers by location, application, cluster, etc.
  • Autonomous, which will migrate guests among hosts.
  • Support for running different command in languages such us shell, powershell, node, python, ruby, java, among others.

  • Scheduling mechanism to schedule tasks and notify admins via email, sms.

                                    ** Containers**

  • YACOM seeks to provide an API interface to kubernetes as well as github, making it easy for developers to obtain resources within the enterprise.

  • Kubernetes under the covers will manage the containers.
  • It will apply the best practices in case of failure, by either doing it autonomically or consulting the sysadmin.
